# 架電時
## 目的
- 現在の検討状況等のヒアリングを通じ、改めて案件化できるするか見極める
- 案件化する場合は、下記ヒアリング項目を確認したうえでFSへ取り次ぎ
- 進捗ない場合は、案件化する条件を把握しHubSpot上に記録を残す
- 注意事項
    - 掘り起こしリードのリスト情報をもとに、HubSpotにて状況確認を徹底
        - **既契約者さまや、別の取引担当者が商談中のコンタクトへの重複接触を避けるため**
        - 担当者名と会社名それぞれで検索をかけ、取引状況を確認する
            - まれに、紐付けが行われていないケースあり
            - 担当者名ではヒットしないが、会社名ではヒットする　など
## トーク内容
- **「お世話になっております。私、AIを活用した姿勢分析ツールをご提供しております、シセイカルテの〇〇と申します」**
    - 連絡先が個人携帯ではなく店舗などの固定電話の場合は、前回対応していただいた方への取り次ぎを依頼する
    - ピンときていない場合は、いつ頃連絡をとっていたか、どんな経緯で相談いただいたかを説明して、記憶を呼び起こしてもらう
        - 以前FBから資料請求のお問い合わせをいただいてまして…
        - 〇月の展示会で一度ご案内させていただきまして…
- **「以前、簡単に弊社サービスのご案内をさせていただいたかと思うのですが、その後のご検討状況などいかがかなと思いまして、今回ご連絡させていただきました」**
    - 大抵は、今はまだ検討できる状況にないので～、また時期をみてこちらから連絡しますので～といなされる
    - あまりに迷惑そうにされた場合を除き、もうワンプッシュのため次へ
- **「弊社のサービスなんですが、利用者さまのご要望をもとに毎月定期的なアップデートを行っておりまして…」**
-**「リリースから1年以上経ち（※2020年1月から計算）、さまざまな業種のお客さまにご導入いただくなかで、効果的にご活用いただいている成功事例など、ぜひご紹介させていただければと思っております」**
    - 前回のコンタクト時と比べ、よりコンテンツが進化していることを伝える
        - アップデートによる機能追加
        - 多様な業種での成功事例増加
- 「どこかで改めてご紹介のお時間を頂戴できませんでしょうか？」
    - デモ紹介のアポ取得につなげる
    - 「今週の水～金の午前中でご都合いかがでしょうか」のように、こちらから具体的に日程を挙げることで、獲得率が上がるケースも
    - 盛り上がって前向きに検討してもらえるようなら、トライアル日程を調整する
 ## ヒアリング実施
アポ取得できた場合
- 「ご説明のお時間をいただくにあたり、より有益な情報をご案内できればと思いまして、事前にいくつかお伺いさせていただいてもよろしいでしょうか？」
    - あなたの力になりたいんです！という姿勢を示して、スムーズなヒアリングにつなげる
### ヒアリング項目
ベースはHubSpotのスニペット参照
#### 定量面
事前の情報収集やヒアリングのなかで把握できているようであればスキップ
- 店舗数
- 従業員数
- 費用感
    - 先方の温度感に応じて
        - 価格だけ独り歩きしてトーンダウンしてしまうこともあるため、ある程度温めてから伝えた方がよい
- 業態深堀り
#### 定性面
- 担当者のタイプ
    - 社交的、ビジネス型、職人タイプ　etc…
- 前向き度
- 現状の悩み
    - 新規囲い込み、リピート率up、単価up、スタッフ育成　etc...
 ## ネクストアクションの握り
- 「そうしましたら、改めて30分程度ご説明のお時間をいただければと思うのですが、近いところでご都合よろしい日時ってございますでしょうか」
    - カレンダーでFSさんの空きスケジュールを確認し、日程を調整する。
 ## HubSpotへの記録
- 架電結果に関わらず、「会社」タブが未作成の場合は既存のコンタクトに関連付けし作成（追加）しておく
    - 「コンタクト」のみしかないリードが存在
- 案件化（FS商談）する場合は、さらに「取引」タブを作成（追加）する

- 連絡した相手、成果、スニペットの内容を入力
    - 実権者につながらなかった場合（受付の方が出て、先生は今施術中ですとか）、コールの成果は「不通」とする
- 不振で終わった場合でも、何が要因なのかをしっかりと残しておく
- 「#h」で呼び出す

### コンタクト・会社・取引（プロパティ）更新
- ヒアリングできた情報をもとに、コンタクト情報の追加・更新を行う
    - 担当者の属性（決裁権のある方かどうか）
    - 業種
    - 施設数、従業員数
    - 課題
    - 不足機能
    - 取引ステージ
        - 商談セッティングできた場合は、取引ステージをP1へ（参照：[取引ステージの定義](https://sapeet.esa.io/posts/2093)）
    - 架電ステータス
        - 不通だった場合は「コール（初回～5回目）」に随時ステータス変更
        - 架電回数のルールについては、業務フロー/IS/架電前の「架電ルール」を参照　https://sapeet.esa.io/posts/2742
    - コンタクト担当者/取引担当者
        - 架電者（直近でアプローチした人）に変更
        - 変更が漏れてしまうと、コンタクトとの接触機会が切れてしまうおそれがあるため、変更を徹底すること！
# 架電後
 ## お礼メール送信
-  HubSpotにてメール送信する
- テンプレメールを送付

### アポ獲得できた場合
- テンプレート活用
    - 日程・Zoomコード（URL、ID、パス）
    - 基本的には終話後即対応！
    - 面談まで間が空く場合（一週間超）や前日にはリマインドメールを設定
### 情報提供希望の場合
- 相手の検討ステージに応じてテンプレートより選択
### オプトアウトされた先へのメール対応
- HubSpotにてオプトアウト先へのメールを作成しようとすると、法的根拠を示すようアラートが表示される
- メール送付について相手の同意が得られていれば良いため、法的根拠のプルダウンにて「コンタクトからの自由意志に基づく同意」を選択
- 同意の説明欄には、メール送信に至った経緯を簡記する
